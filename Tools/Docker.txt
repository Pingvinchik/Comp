====== Docker ======
Created Понедельник 29 Май 2017

Альтернатива виртуальной машине (долго загружается, потребляет больше ресурсов).

Решает проблему деплоя (развертывания) приложения.

После упаковки приложения в контейнер, его можно запускать в любом окружении.

Процессы изолируются (конфликтующие версии изолированны друг от друга), ресурсы ограничиваются для каждого контейнера.

Цель не создать изолированное окружение, а запустить приложение.

В контейнер накидываются образы необходимых ресурсов: ос, java, mysql...

Упаковывается в образ
