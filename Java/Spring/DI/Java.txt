====== Java ======

Помечая класс @Component мы говорим, что Spring должен будет создать для этого класса bean.

Чтобы Spring просканировал классы в поисках данной аннотации мы должны указать это или в Java Configuration классе, либо в xml.

Для этого создаем класс с аннотацией @Configuration и указываем аннотацию @ComponentScan. Таким образом будут сканироваться классы в том же package, что и конфигурационный класс.

Можно разбить конфигурационный класс на несколько и объединить их @Import

Чтобы добавить бины в контейнер, необходимо в JavaConfig добавить метод, который возвращает экземлпяр класса и пометить его @Bean. Тем самым, Spring зарегистрирует его как бин. Можно даже передать в этот метод бин и Spring сам подставит его.

Теперь для того чтобы Inject bean необходимо пометить @Autowired либо поле, либо конструкор, либо setter.

Для бинов, которые необходимо часто менять свойства лучше прописывать в xml чтобы не пересобирать.
