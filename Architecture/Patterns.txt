====== Patterns ======
Created Воскресенье 09 Апрель 2017

Использовать enum вместо множества String



1. Инкапсулировать те части, которые подвержены изменениям.
2. Программировать на уровне интерфейсов, а не реализаций. Чтобы можно было на лету подменять экземпляры.
3. Предпочитать композицию наследованию. Наследование ограничивает повторное использование кода класса, например, если какой-либо класс наследует что-то он не сможет занаследовать новый класс. Т.е. не получится добавить функционал к классу если он уже наследует что-то. В отличии от композиции, поведение объекта устанавливается во время компиляции, а в композиции может меняться в рантайме.
4. Loose coupling (низкая связанность) между взаимодействующими классами.
5. Класс должен быть открыт к расширению, но закрыт к модификациям.
6. Поля класса не должны указывать на конкретный объект, создаваться тоже не через конкретный.
7. Класс не должен обращаться к слишком большому числу объектов (должен знать о других объектах как можно меньше). Т.е. обращаться только к: полям класса, переданным в качестве параметра или если сам этот метод создал объект. НО не вызывать методы у объекта полученым через return другого метода
